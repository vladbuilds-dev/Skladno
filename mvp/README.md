# Складно MVP — демо-конвейер

Прогноз спроса → MILP-оптимизация → план поставки в xlsx.
Это ядро консьерж-MVP: в проде на вход идут выгрузки API WB, здесь — демо-CSV того же формата.

## Запуск

```bash
pip install numpy scipy pandas pulp openpyxl
python3 generate_demo_data.py     # создаёт demo_data/ (продажи, остатки, коэффициенты, тарифы)
python3 skladno_mvp.py            # решает и пишет план_поставки.xlsx
python3 skladno_mvp.py --budget 800000 --truck 2500   # свои ограничения
```

## Что внутри

1. **Прогноз** — иерархический: ставка SKU × доли регионов портфеля, эмпирико-байесовский бленд по ячейкам (валидирован симуляцией: даёт +21% к оракулу против +52% у наивного per-cell). Три сценария спроса P25/P50/P75.
2. **MILP (PuLP/CBC)** — переменные: сколько каждого SKU на какой склад и в какую дату. Учитывает: календарь коэффициентов приёмки (закрытые дни исключены), бюджет закупки, вместимость фуры на направление, тарифы хранения + заморозку капитала, надбавку за нелокальность 2,5% и потерю продаж при обслуживании издалека, штраф за поздний завоз. Решается за секунды на 40 SKU × 8 складов × 14 дат.
3. **Выгрузка** — `план_поставки.xlsx`: лист «План поставки» (склад, дата, баркод, количество — формат загрузки WB) и лист «Экономика» (сравнение с baseline «всё на Коледино»).

## Результат на демо-данных

Продавец: 40 SKU, оборот ~5 млн ₽/мес. План: 8 складов, даты подобраны под низкие коэффициенты. Логистические издержки −86% против «всё на Коледино» (40 тыс. против 287 тыс. ₽/мес), ожидаемая прибыль +352 тыс. ₽/мес.

**Честная оговорка:** демо-календарь делает Коледино дорогим (коэф. 3–8), поэтому цифра экономии выше реалистичных 15–20% логистического бюджета. На реальных данных клиента разброс будет зависеть от его текущей практики — это и есть то, что проверяет консьерж-MVP.

## Путь к проду

- Заменить `demo_data/` на клиенты API WB: `/orders` (спрос по регионам), `warehouseRemains` (остатки), `/acceptance/coefficients` (календарь), API тарифов.
- Пересчёт при изменении календаря коэффициентов (обновляются каждые несколько часов) + топ-3 альтернативных плана с порогами («грузить на Тулу при К≤2»).
- CBC → HiGHS при росте размерности (200+ SKU).
