Складно. demo MVP
40 SKU · 8 складов · CBC/MILP · Optimal

План поставки посчитан. +352 тыс ₽/мес к прибыли против «всё на Коледино»

Демо-прогон конвейера на данных продавца с оборотом ~5 млн ₽/мес: выгрузка продаж → прогноз спроса → оптимизация → готовый план в формате загрузки Wildberries. Реальный решатель, реальная модель — демо-данные.

Как считается

01 · данные
Выгрузка API WB

60 дней продаж по регионам (/orders), остатки FBO, календарь коэффициентов на 14 дней, тарифы хранения.

02 · прогноз
Спрос SKU × регион

Иерархическая модель: ставка SKU × доли регионов портфеля, байесовский бленд. Три сценария: P25 / P50 / P75.

03 · оптимизация
MILP одной задачей

Что, сколько, куда и на какую дату — при бюджете закупки, фуре, коэффициентах, хранении и нелокальности.

04 · план
Файл для WB

Excel в формате загрузки: склад, дата, баркод, количество. Плюс экономика решения по статьям.

Календарь коэффициентов приёмки

То, что делает задачу нетривиальной: коэффициенты меняются по складам и датам, часть дней закрыта. Белой рамкой — даты, выбранные оптимизатором.

×0 — бесплатно ×1–2 ×3–4 ×5+ приёмка закрыта выбрано планом

План поставки

8 направлений, даты подобраны под низкие коэффициенты с учётом штрафа за поздний завоз. Клик по складу фильтрует таблицу.

Экономика за 30 дней

Ожидание по сценариям спроса. Baseline — типичная практика: весь объём на Коледино в ближайший открытый день.

Честная оговорка. В демо-календаре Коледино дорогой (коэф. 3–8), поэтому экономия −86% по логистике выше типичных 15–20% логистического бюджета. Цель консьерж-MVP — получить эту цифру на реальных данных клиента.
